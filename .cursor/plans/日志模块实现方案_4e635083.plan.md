# 日志模块实现方案

## 目标

实现一个统一的日志模块，记录每个session_id的所有会话信息（用户消息和AI回复），存储到日志文件中，便于问题定位。

## 架构设计

### 日志模块结构

```
services/
  └── logger_service.py  # 日志服务模块（新建）
```

### 日志配置

- **日志级别**: INFO及以上（INFO, WARNING, ERROR）
- **日志格式**: 单个日志文件，所有session的日志混合存储
- **日志内容**: 所有用户消息和AI回复，以及关键操作信息
- **日志格式**: 包含时间戳、session_id、日志级别、消息内容等

### 实现要点

1. **创建日志服务模块** (`services/logger_service.py`)

   - 使用Python标准库`logging`模块
   - 配置文件处理器（FileHandler）和格式化器
   - 提供按session_id记录会话信息的方法
   - 支持记录用户消息、AI回复、工具调用等关键信息

2. **日志格式设计**

   - 时间戳 | session_id | 级别 | 类型 | 内容
   - 例如: `2024-01-01 10:00:00 | session_123 | INFO | USER_MSG | 用户消息内容`
   - 例如: `2024-01-01 10:00:01 | session_123 | INFO | AI_REPLY | AI回复内容`

3. **集成到现有服务**

   - 在`agent_service.py`中集成日志记录
   - 在`session_manager.py`中记录session创建和销毁
   - 记录所有用户输入和AI输出

4. **日志文件管理**

   - 日志文件路径: `logs/agent_{日期}.log` 或 `logs/agent.log`
   - 支持日志轮转（可选，避免文件过大）

## 实现步骤

1. 创建`services/logger_service.py`日志服务模块

   - 初始化日志配置
   - 提供`log_session_message()`方法记录会话消息
   - 提供`log_operation()`方法记录关键操作

2. 修改`services/agent_service.py`

   - 导入日志服务
   - 在处理用户消息时记录日志
   - 在生成AI回复时记录日志
   - 在调用工具时记录日志

3. 修改`services/session_manager.py`

   - 记录session创建和销毁
   - 记录session状态变化

4. 创建日志目录

   - 确保`logs/`目录存在
   - 添加`.gitignore`规则（如需要）

## 日志记录的关键点

- **用户消息**: 记录完整的用户输入
- **AI回复**: 记录完整的AI响应
- **工具调用**: 记录工具名称、参数、结果
- **错误信息**: 记录异常堆栈信息
- **Session生命周期**: 记录session创建、销毁、状态变化
